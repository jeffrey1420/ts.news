# typescript.news Site Contract

This repo is a Nuxt 4 and Nuxt Content 3 news site. Articles live in locale folders:

- `content/en/articles/`
- `content/fr/articles/`
- `content/de/articles/`

Each article slug should exist in all three locale folders. Routes are `/articles/slug`, with locale handling done by the app.

## Article Schema

Defined in `content.config.ts`:

- `title`: string, required.
- `description`: string, required.
- `date`: string, required, `YYYY-MM-DD`.
- `author`: optional string, use `lschvn`.
- `tags`: optional string array, use canonical slugs.
- `readingTime`: optional number, usually omit.
- `image`: optional string, use generated local hero path.
- `tldr`: optional string array, expected for new articles.
- `faq`: optional array of `{ question, answer }`, expected for new articles.

The article page renders TLDR, FAQ accordions, NewsArticle JSON-LD, FAQPage JSON-LD, related articles by shared tags, and raw Markdown alternates.

## Canonical Tags

Use only:

- `typescript`
- `javascript`
- `runtimes`
- `tooling`
- `frameworks`
- `css`
- `ai`
- `security`
- `performance`
- `ecosystem`

The first tag becomes the listing badge, so put the primary angle first.

## Hero Images

Hero images are generated by `scripts/generate-images.ts` from the static `HEROES` map. New articles must add a map entry, run `bun scripts/generate-images.ts`, and reference:

```yaml
image: "/images/heroes/YYYY-MM-DD--descriptive-slug.png"
```

## House Style

- Audience: developers who ship TypeScript, JavaScript, and web platform code.
- Tone: direct, specific, technically literate, lightly opinionated.
- Use inline source links for important claims.
- Avoid em dash U+2014.
- Avoid generic AI prose and padding.
- Prefer concrete impact: compatibility, migration, performance, security, API changes, developer workflow, ecosystem consequences.

## Validation Commands

```bash
bun scripts/consolidate-tags.ts --dry
bun run lint
bun run typecheck
bun run generate
```
