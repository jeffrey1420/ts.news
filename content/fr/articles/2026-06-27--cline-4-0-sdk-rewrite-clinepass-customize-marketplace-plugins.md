Cline a publié [v4.0.0 le 26 juin 2026](https://github.com/cline/cline/releases/tag/v4.0.0), une version majeure qui passe de v3.89.2 (11 juin) et redéfinit ce qu'est Cline. Là où la ligne 3.x était une extension VS Code qui a vu grandir une CLI à ses côtés, la 4.0 fait du [SDK Cline](https://docs.cline.bot/cline-sdk/overview) partagé le moteur de tout, et y porte l'extension VS Code. Cline se décrit désormais comme « l'agent de code open source dans votre IDE et votre terminal », et le README est explicite sur le fait que le SDK est « le même moteur qui fait tourner la CLI, Kanban, l'extension VS Code et le plugin JetBrains ». La réécriture de l'extension fait le titre ; les couches d'abonnement, de marketplace et de plugins sont ce qui transforme la réécriture en un pari de plateforme.

## La migration SDK : un seul moteur pour quatre surfaces

Le changement central de la 4.0 est que l'extension VS Code ne fait plus tourner sa propre implémentation de tâches historique. Les tours d'agent, les outils, la coordination du mode Plan/Act, MCP, les checkpoints, la télémétrie, les changements de fournisseur, la compaction, les limites d'erreurs et l'historique des tâches passent désormais par la couche de session partagée du SDK. Le workflow de build et de packaging de l'extension migre aussi vers [Bun](https://bun.sh), que Cline utilise déjà dans le reste du dépôt.

Le gain d'architecture est la cohérence. La CLI Cline (désormais sur sa propre ligne v3.0.x, à [cli-v3.0.31](https://github.com/cline/cline/releases/tag/cli-v3.0.31) au 27 juin), le tableau de tâches [Kanban](https://github.com/cline/kanban) pour agents en parallèle, le plugin JetBrains et l'extension VS Code partagent tous un seul runtime, donc une correction de comportement ou un changement d'outil atterrit sur toutes les surfaces au lieu d'être réimplémenté par client. La gestion Plan/Act est refondue via des coordinateurs SDK avec une meilleure parité CLI et une continuation automatique lors du passage de Plan à Act. L'exécution terminal est simplifiée via le chemin run-commands du SDK, avec un formatage structuré de commande plus sûr et des indications non interactives plus claires. Cette même logique de consolidation est ce qui a poussé OpenAI à bâtir une plateforme gouvernable autour de [Codex 0.142](/articles/2026-06-27--openai-codex-0-142-token-budgets-multi-agent-web-search) plus tôt cette semaine, et le schéma plus large des agents de code passant d'outils à surface unique à des plateformes SDK-first est désormais la forme par défaut de la catégorie.

## ClinePass : un abonnement géré dans l'extension

Le changement produit le plus visible est ClinePass. C'est un flux d'abonnement et d'onboarding intégré à l'extension VS Code, couvrant la sélection de fournisseur, l'inscription et la prise en charge d'abonnement, les listes de modèles en direct, les états d'erreur d'organisation et de droits, les alertes de crédit épuisé, ainsi qu'une gestion d'authentification et d'erreur ClinePass plus claire. Des réglages génériques de fournisseur SDK et la prise en charge du catalogue de modèles permettent à davantage de fournisseurs de partager le même sélecteur de modèles, contrôles de raisonnement, identifiants de modèles dynamiques et gestion de modèles personnalisés.

Lu simplement, Cline livre désormais un chemin géré : s'abonner, choisir un modèle, lancer. C'est un tournant pour un projet qui a bâti son audience sur « apportez votre propre clé API », et c'est le même tournant que le reste de la catégorie a pris à mesure que les agents de code passaient d'outils de passionnés à des produits gérés. Les fournisseurs BYO-key ne sont pas supprimés ; ils côtoient le flux géré, et le nouveau `providers.json` plus le catalogue de modèles partagé sont conçus pour préserver les réglages entre les changements de fournisseur.

## Le marketplace Customize et les Cline Plugins

Le marketplace Customize regroupe les Skills, les serveurs MCP et les Plugins dans une seule interface, avec des onglets installé et marketplace, une recherche et un filtrage, des flux d'installation et de désinstallation, ainsi que des contrôles d'activation et de désactivation. Il prend aussi en charge les skills groupées dans des plugins et peut être réorganisé sous un point d'entrée Customize.

Les [Cline Plugins](https://github.com/cline/cline/releases/tag/v4.0.0) sont la nouvelle primitive d'extensibilité. Ils permettent d'étendre Cline avec des outils, workflows, skills et capacités basés sur MCP sur mesure pour une équipe ou un projet, et peuvent s'installer en plein milieu d'une session. La prise en charge MCP s'étend désormais aux plugins, et le hub MCP se rafraîchit automatiquement après une installation depuis le marketplace, de sorte qu'un serveur nouvellement installé est disponible sans redémarrage manuel. L'onglet MCP Marketplace peut aussi être désactivé à distance pendant que les serveurs MCP installés restent accessibles. C'est le même vecteur d'extensibilité sur lequel Claude Code, Codex et la [version opencode desktop](/articles/2026-04-19--opencode-desktop-electron-tauri-typescript) convergent : une couche de plugins et de MCP qui transforme un agent de code en quelque chose que les équipes peuvent personnaliser sans le forker.

## Prompts en file d'attente, refonte des fournisseurs et défauts plus sûrs

Un lot d'améliorations visibles par les développeurs complète la version. Les prompts en file d'attente corrigent une frustration de longue date : les messages soumis pendant que Cline travaille sont désormais mis en file, s'affichent pendant que le tour en cours se diffuse, et peuvent être annulés avant de s'exécuter, au lieu d'être abandonnés ou d'entrer en concurrence avec le tour actif. L'édition et la régénération permettent de modifier un message utilisateur précédent et de relancer depuis ce point, avec Escape qui annule l'édition localement et des libellés Reset Chat et Reset Code plus clairs.

La configuration des fournisseurs et des modèles est refondue autour de `providers.json`, du catalogue de modèles et de la configuration de session SDK, et la version câble un large ensemble de fournisseurs et de modèles : les modèles ClinePass, Fireworks GLM 5.2, Kimi K2.6 Fast, [Kimi K2.7 Code](/articles/2026-06-12--kimi-k2-7-code-mimo-code), Qwen 3.7 Plus, MiniMax M3, SAP AI Core, la récupération de modèles LiteLLM, les identifiants Codex OAuth et les réglages de modèles compatibles OpenAI. Les composants de réglages génériques partagés et les sélecteurs de raisonnement remplacent de nombreuses vues spécifiques à un fournisseur.

Les défauts de sécurité se resserrent. L'auto-approbation des commandes est désormais désactivée par défaut pour les configurations nouvelles et réinitialisées, et l'interface d'auto-approbation est simplifiée. La version corrige aussi le budgeting des résultats d'outils SDK et des messages fournisseur en tronquant les grandes sorties d'outils par défaut, en plafonnant le texte de l'assistant, en limitant l'ingestion des sorties bash, lecture de fichier et recherche, en bornant les budgets média et en groupant les réécritures de lectures obsolètes pour préserver les caches de préfixe des fournisseurs. La fiabilité terminal reçoit une longue liste de corrections, dont la capture de sortie Windows autonome, un traitement PowerShell renforcé, la coalescence des heredocs et la suppression des échos de commande dupliqués.

## Ce qu'il faut surveiller

Deux réserves méritent d'être signalées avant la mise à niveau. Les sous-agents sont temporairement désactivés dans l'extension VS Code pendant la stabilisation de l'expérience basée sur le SDK, donc les équipes qui comptaient sur l'[orchestration multi-agent](/articles/2026-06-22--sakana-fugu-multi-agent-orchestration-frontier) via la fonctionnalité de sous-agent de Cline devront attendre son retour. La fonctionnalité historique Explain Changes est supprimée dans le cadre du nettoyage de la migration SDK. Tout le reste est additif. Le changelog complet entre v3.89.2 et v4.0.0 est sur la [vue de comparaison](https://github.com/cline/cline/compare/v3.89.2...v4.0.0), et la ligne CLI continue indépendamment sur la branche v3.0.x. Cline compte environ 64 000 étoiles GitHub et se situe dans une catégorie qui se consolide vite ; la 4.0 est la version où l'outil cesse d'être une extension et devient une plateforme.
